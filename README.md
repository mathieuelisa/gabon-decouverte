## Convention de nommage des branches

Pour assurer une bonne organisation et une lecture claire de l’historique Git, chaque branche doit suivre une convention de nommage standardisée.  
Cette convention permet d’identifier rapidement **le type de travail effectué** (nouvelle fonctionnalité, correction, mise à jour technique, etc.) et de garder un dépôt propre et cohérent.

Voici un exemple :

- `feature/add-activity-listing` pour une nouvelle fonctionnalité
- `fix/reservation-form-error` pour une correction de bug
- `style/update-homepage-banner` pour un ajustement de design
